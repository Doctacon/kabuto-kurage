# Project Purpose

This repository is a learning codebase. Its purpose is to help Chris build hands-on familiarity with data-platform and infrastructure concepts that may matter for a Staff Data Engineer interview process at Jellyfish.

The project should prioritize practical, local-first exercises over production completeness. Useful learning targets include:

- Infrastructure as Code, especially Terraform-style resource definition and state.
- Multi-tenant data architecture and customer-data isolation patterns.
- Large-scale ingestion patterns for external SaaS sources such as GitHub and Jira.
- Data orchestration with Dagster or an equivalent asset-aware orchestrator.
- Lakehouse table formats such as Delta Lake or Apache Iceberg.
- Observability for ingestion and transformation pipelines.

The codebase should remain aligned with the project's open-source-first principle. Prefer local, open-source, self-hostable tooling where feasible.
